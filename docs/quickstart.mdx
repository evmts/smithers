# Quick Start

This guide will walk you through installing Smithers and creating your first AI agent.

## Prerequisites

- Node.js 18+ or Bun
- An Anthropic API key

## Installation

<CodeGroup>

```bash npm
npm install smithers
```

```bash bun
bun add smithers
```

```bash pnpm
pnpm add smithers
```

```bash yarn
yarn add smithers
```

</CodeGroup>

## Set Up Your Environment

Create a `.env` file with your Anthropic API key:

```bash
ANTHROPIC_API_KEY=sk-ant-...
```

## Create Your First Agent

Create a file called `review-agent.tsx`:

```tsx
import { Claude, Constraints, OutputFormat } from 'smithers'

export function ReviewAgent({ files }) {
  return (
    <Claude tools={[filesystem, grep]}>
      <Constraints>
        - Focus on bugs and security issues
        - Be constructive, not pedantic
        - Ignore formatting (that's what linters are for)
      </Constraints>

      Review these files for issues: {files.join(', ')}

      <OutputFormat>
        Return JSON: {"{"} issues: [{"{"} file, line, severity, message {"}"}] {"}"}
      </OutputFormat>
    </Claude>
  )
}
```

## Run the Agent

<Steps>
  <Step title="Preview the plan">
    ```bash
    smithers plan review-agent.tsx --props '{"files": ["src/auth.ts"]}'
    ```

    This shows you the XML plan without executing:

    ```xml
    <claude tools="filesystem,grep">
      <constraints>
        - Focus on bugs and security issues
        - Be constructive, not pedantic
        - Ignore formatting (that's what linters are for)
      </constraints>

      Review these files for issues: src/auth.ts

      <output-format>
        Return JSON: { issues: [{ file, line, severity, message }] }
      </output-format>
    </claude>
    ```
  </Step>

  <Step title="Execute with approval">
    ```bash
    smithers run review-agent.tsx --props '{"files": ["src/auth.ts"]}'
    ```

    You'll see the plan and be prompted to approve before execution.
  </Step>

  <Step title="Auto-approve for CI/CD">
    ```bash
    smithers run review-agent.tsx --props '{"files": ["src/auth.ts"]}' --auto-approve
    ```

    Skip the approval prompt for automated workflows.
  </Step>
</Steps>

## Project Scaffolding

Use the init command to create a new project with all the boilerplate:

```bash
smithers init my-agent-project
cd my-agent-project
```

This creates:

```
my-agent-project/
  package.json       # Dependencies configured
  tsconfig.json      # TypeScript with JSX support
  .env.example       # API key template
  src/
    agent.tsx        # Starter agent template
```

## Understanding the Output

When you run an agent, Smithers displays:

1. **The Plan**: The XML representation of your agent's intent
2. **Approval Prompt**: Confirm before execution (unless `--auto-approve`)
3. **Execution Log**: Real-time progress as Claude runs
4. **Result**: The final output from your agent

```
┌─ Plan ─────────────────────────────────────────────────────┐
│                                                            │
│  <claude tools="filesystem,grep">                          │
│    <constraints>                                           │
│      - Focus on bugs and security issues                   │
│      - Be constructive, not pedantic                       │
│    </constraints>                                          │
│    Review these files for issues: src/auth.ts              │
│  </claude>                                                 │
│                                                            │
└────────────────────────────────────────────────────────────┘

? Execute this plan? (Y/n) y

Executing...
[Frame 1] Running <claude>
[Frame 1] Complete

Result:
{
  "issues": [
    { "file": "src/auth.ts", "line": 42, "severity": "high", "message": "..." }
  ]
}
```
