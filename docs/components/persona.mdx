# Persona

```tsx
import { Persona } from "smithers-orchestrator";

<Claude>
  <Persona role="security expert">
    You specialize in application security and OWASP Top 10.
  </Persona>
  Review this authentication code.
</Claude>
```

<Info>
Persona content serializes into the prompt as a `<persona>` element.
</Info>

## Props

| Prop | Type | Description |
|------|------|-------------|
| `role` | `string?` | Short role name (e.g., "security expert") - optional |
| `children` | `ReactNode` | Detailed expertise description |

## Examples

### Code Reviewer

```tsx
<Claude>
  <Persona role="senior code reviewer">
    Focus on correctness, performance, maintainability.
    Be constructive but thorough.
  </Persona>
  Review this pull request.
</Claude>
```

### Domain Expert

```tsx
<Claude>
  <Persona role="database specialist">
    Expert in query optimization, indexing, schema design.
  </Persona>
  Analyze these slow queries.
</Claude>
```

### With Constraints

```tsx
<Claude>
  <Persona role="technical writer">
    You write clear, concise documentation.
  </Persona>
  <Constraints>
    - Use simple language
    - Include code examples
  </Constraints>
  Document this API endpoint.
</Claude>
```

## Best Practices

- **Be specific:** "React performance specialist" vs "expert developer"
- **Include behavior:** "Always check for understanding before moving on"
- **Match task:** code review → "meticulous reviewer", debugging → "systematic debugger"
