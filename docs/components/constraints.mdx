# Constraints

```tsx
import { Constraints } from "smithers-orchestrator";

<Claude>
  <Constraints>
    - Keep responses under 500 words
    - Use bullet points for lists
    - Include code examples
  </Constraints>
  Explain the SOLID principles.
</Claude>
```

<Info>
Constraints serialize into the prompt as a `<constraints>` element.
</Info>

## Props

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | Rules, typically as a list |

## Examples

### Output Format

```tsx
<Claude>
  <Constraints>
    - Respond in JSON format only
    - Include "summary" and "issues" fields
  </Constraints>
  Analyze this code.
</Claude>
```

### Behavioral

```tsx
<Claude>
  <Constraints>
    - Only modify files in src/components
    - Run tests after each change
    - Commit with descriptive messages
  </Constraints>
  Refactor the Button component.
</Claude>
```

### With Persona

```tsx
<Claude>
  <Persona role="security engineer">
    You specialize in application security.
  </Persona>
  <Constraints>
    - Focus on OWASP Top 10
    - Rate issues by severity
  </Constraints>
  Audit the auth module.
</Claude>
```

## Best Practices

- **Be specific:** "Keep functions under 50 lines" vs "write clean code"
- **Use positive framing:** "Use parameterized queries" vs "don't concatenate SQL"
- **Group by category:** Code Quality, Testing, Performance
