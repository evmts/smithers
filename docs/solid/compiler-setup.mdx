# Solid Compiler Setup

This guide shows how to compile JSX so it targets the `solid-smithers` renderer instead of the DOM.

## Babel

```json
{
  "presets": [
    [
      "babel-preset-solid",
      {
        "moduleName": "solid-smithers",
        "generate": "universal"
      }
    ]
  ]
}
```

## Vite

```ts
import { defineConfig } from "vite"
import solid from "vite-plugin-solid"

export default defineConfig({
  plugins: [
    solid({
      solid: {
        moduleName: "solid-smithers",
        generate: "universal"
      }
    })
  ]
})
```

## TypeScript

```json
{
  "compilerOptions": {
    "jsx": "preserve",
    "jsxImportSource": "solid-smithers"
  }
}
```

Or per-file pragma:

```tsx
/** @jsxImportSource solid-smithers */
```

## Expected Exports from solid-smithers

`solid-smithers` should export the Solid universal renderer surface:

- `render`
- `effect`
- `memo`
- `createComponent`
- `createElement`
- `createTextNode`
- `insertNode`
- `insert`
- `spread`
- `setProp`
- `mergeProps`
- `use`

Also re-export Solid control flow:

- `For`, `Show`, `Switch`, `Match`, `Index`, `ErrorBoundary`

And Smithers components:

- `Claude`, `Phase`, `Step`, `Subagent`, `Persona`, `Constraints`, `OutputFormat`, `Human`, `Stop`, `Output`, `File`, `Worktree`

## Sanity Check

Compile a tiny app and confirm:

- JSX output calls into `solid-smithers`.
- `renderPlan()` returns XML identical to React output.
